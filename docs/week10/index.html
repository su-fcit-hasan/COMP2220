  • Client-side scripting is a type of computer programming that is executed by the user's web browser. It is a type of code that is executed by the web browser and the user will see the results. An example of client-side scripting is JavaScript.
  • Server-side scripting is a type of computer programming that is executed on the server. It is a type of code that is executed by the server and the user will not see the results. An example of server-side scripting is PHP.

    What are the main limitations of the JavaScript as compared to server side scripting languages like PHP?
    • JavaScript is a client-side scripting language, which means that it is executed on the user's web browser. This means that the user can see the code and manipulate it. This can be a security risk, as the user can see the code and potentially exploit it. Server-side scripting languages like PHP are executed on the server, which means that the user cannot see the code and cannot manipulate it. This makes server-side scripting languages more secure than JavaScript.
    • JavaScript is limited to the capabilities of the user's web browser. This means that JavaScript can only do what the web browser allows it to do. Server-side scripting languages like PHP are not limited by the capabilities of the user's web browser, which means that they can do more than JavaScript.
    • JavaScript is not as fast as server-side scripting languages like PHP. This is because JavaScript is executed on the user's web browser, which means that it is limited by the speed of the user's web browser. Server-side scripting languages like PHP are executed on the server, which means that they are not limited by the speed of the user's web browser.

    Excercise 3 - week10

    Describe the main responsibilities of a Web Server.

    A web server is a computer program that is responsible for accepting HTTP requests from clients, such as web browsers, and serving them with HTTP responses, usually with HTML pages. The main responsibilities of a web server are to store, process and deliver web pages to clients.

    • A web server is responsible for storing web pages and other web content, such as images, videos, and other files. This content is stored on the web server's hard drive or other storage devices.
    • A web server is responsible for processing web pages and other web content. This means that the web server is responsible for executing server-side scripts, such as PHP, and generating dynamic web pages.
    • A web server is responsible for delivering web pages and other web content to clients. This means that the web server is responsible for sending web pages and other web content to clients when they request it.
